# Feature Execute — 任务执行器

逐任务执行 `tasks.md` 中的代码改动，每完成一个任务必须通过门控验证才能继续。适合单任务精确执行场景；批量自动执行请使用 `rc:plan-executor`。

## 工作流程

### Step 1: 加载上下文

1. **读取 CLAUDE.md**：了解项目架构边界、代码规范
2. **读取 CLAUDE.md Verification 章节**：获取当前项目的验证命令（lint / typecheck / test）
3. **读取 tasks.md**：定位 `docs/features/{module}/{version}/tasks.md` 或 `docs/features/{module}/{version}/{side}/tasks.md`
4. **读取 design.md**：搜索同目录下的 design.md（与 tasks.md 同级），理解设计意图，避免偏离设计
5. **搜索 `docs/solutions/`**：查找相关的已有解决方案和经验教训

### Step 2: 找到下一个待执行任务

1. 如果用户指定了 task-number，定位到该任务
2. 否则，找到第一个 ⬜ 状态的任务
3. 检查该任务的前置依赖是否都已完成（✅）
4. 如果前置依赖未完成，提醒用户并停止

### Step 3: 执行单个任务

#### 3.1 标记执行中

将任务状态从 `⬜ 待处理` 改为 `🔧 执行中`。

#### 3.2 实现代码变更

按照任务描述中的子任务列表和代码骨架，逐步实现：

1. **读取目标文件**（修改时）或检查父目录（新建时）
2. **按代码骨架实现完整代码**：骨架是指导，不是限制；但不要偏离设计意图
3. **确保 import 正确**：新增文件的导入、现有文件的引用都要正确
4. **风格一致**：遵循项目 CLAUDE.md 中的代码规范
5. **逐个子任务标记完成**

#### 3.3 标记完成

将任务状态从 `🔧 执行中` 改为 `✅ 已完成`。

### Step 4: 增量验证（强制门控）

**此步骤不可跳过。**

从项目 CLAUDE.md 的 Verification 章节读取对应变更类型的验证命令，然后执行：

1. **编译/类型检查**：必须通过
2. **测试**：已有测试不能回归（新增测试失败需要修复）
3. **架构检查**（如果有）：模块依赖方向不能违反

#### 验证通过

报告结果，询问用户是否继续下一个任务。

#### 验证失败

1. 分析错误信息，尝试自行修复（最多 2 次）
2. 如果修复后验证通过，继续
3. 如果 2 次修复后仍失败：
   - 将任务状态改为 `⚠️ 验证未通过`
   - 在任务详情中记录错误信息
   - **停止执行**，向用户报告失败原因
   - 不要跳过失败任务继续执行下一个

### Step 5: 继续或停止

- 如果用户要求继续，回到 Step 2 处理下一个任务
- 如果用户要求停止，汇报当前进度
- 当所有任务完成后，输出完整进度摘要

## 执行原则

### 做

- 严格按照 tasks.md 中的代码骨架实现
- 读取目标文件后再修改，理解上下文
- 每个任务完成后立即更新 tasks.md 状态
- 遇到设计与代码现状冲突时，先停下来向用户确认

### 不做

- ❌ 不越界：不实现 tasks.md 红线列表中的功能
- ❌ 不自由发挥：不添加 tasks.md 未要求的功能
- ❌ 不跳过验证：每个任务完成后必须通过门控验证
- ❌ 不跳过失败：验证失败就停止，不盲目继续
- ❌ 不修改 design.md：执行阶段不改设计，有问题先提出

## 状态标记

| 标记 | 含义 |
|------|------|
| ⬜ | 待处理 |
| 🔧 | 执行中 |
| ✅ | 已完成 |
| ⚠️ | 验证未通过 |

## 与其他 Skill 的关系

```
rc:feature-plan → tasks.md → rc:feature-execute（逐个任务）
                           → rc:plan-executor（批量自动）
```

- `rc:feature-execute` 适合需要精确控制每个任务的场景
- `rc:plan-executor` 适合信任自动化批量执行的场景

## 原则

1. **一次一个任务**：不要同时修改多个任务的文件，保持原子性
2. **验证是门控**：不通过验证就不能标记完成，不能继续下一个
3. **状态即进度**：tasks.md 的状态标记是唯一的进度真相源
4. **尊重红线**：tasks.md 中 ❌ 标记的功能绝对不实现
5. **验证命令从 CLAUDE.md 读取**：不硬编码验证命令，从 CLAUDE.md Verification 章节动态获取
6. **用中文输出**：所有状态报告使用中文
